## ğŸ“ Key Learning Points

### 1. Command (Write)
- **Purpose**: Change application state
- **Example**: CreateAlarmCommand
- **Handler**: Executes business logic, saves to write DB, publishes events
- **Database**: PostgreSQL (ACID, normalized)

### 2. Query (Read)
- **Purpose**: Retrieve data without side effects
- **Example**: GetAlarmsQuery
- **Handler**: Fetches from read DB
- **Database**: MongoDB (fast, denormalized)

### 3. Event (Sync)
- **Purpose**: Keep read model in sync with write model
- **Example**: AlarmCreatedEvent
- **Handler**: Updates materialized view in MongoDB
- **Pattern**: Eventually consistent

### 4. Separation Benefits
- Different databases for different purposes
- Read operations don't slow down writes
- Write operations don't slow down reads
- Can scale independently
- Clear code organization

## ğŸ” Comparison: Before vs After CQRS

### Before (Traditional)
```typescript
@Injectable()
export class AlarmsService {
  create(dto) {
    const alarm = this.factory.create(dto);
    return this.repository.save(alarm);  // Same DB for everything
  }
  
  findAll() {
    return this.repository.findAll();     // Same DB for everything
  }
}
```

### After (CQRS)
```typescript
@Injectable()
export class AlarmsService {
  create(command) {
    return this.commandBus.execute(command);  // â†’ Write DB
  }
  
  findAll() {
    return this.queryBus.execute(new GetAlarmsQuery());  // â†’ Read DB
  }
}
```
